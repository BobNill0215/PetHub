# 开发计划

## 一、总体路线图

```
Phase 1 ──── MVP 核心闭环 ──── 8-10周
   │
   ▼
Phase 2 ──── 社区深化 ──────── 4-6周
   │
   ▼
Phase 3 ──── 商业闭环 ──────── 6-8周
   │
   ▼
Phase 4 ──── 增长与运营 ────── 4-6周
   │
   ▼
持续迭代 ──── 数据驱动优化 ──── 长期
```

---

## 二、Phase 1：MVP 核心闭环（第1-10周）

**目标**：快速上线可用版本，验证核心价值

### 第1-2周：项目初始化

| 任务 | 产出 |
|------|------|
| 技术栈搭建 | 前后端项目脚手架、CI/CD 流水线 |
| PostgreSQL + MongoDB + Redis 部署 | Docker 本地 + 云环境 |
| 用户认证系统 | 注册/登录/Token/JWT |
| 基础 UI 框架 | 通用组件库、布局、主题 |

### 第3-4周：用户与宠物档案

| 任务 | 产出 |
|------|------|
| 用户注册/登录页面 | 手机号 + 验证码、密码 |
| 个人主页 | 头像/昵称/简介编辑 |
| 宠物档案 CRUD | 添加/编辑/删除宠物 |
| 宠物品种数据 | 预设品种库（猫/狗） |

### 第5-7周：内容社区

| 任务 | 产出 |
|------|------|
| 图文动态发布 | 拍照/相册选择 + 文字描述 + 标签 |
| 信息流展示 | 关注流 + 推荐流 |
| 互动功能 | 点赞/评论/收藏 |
| 图片上传 + OSS 集成 | 压缩/裁剪/CDN |
| 话题标签 | #话题 创建与聚合 |

### 第8-10周：基础交易 + 管理后台

| 任务 | 产出 |
|------|------|
| C2C 商品发布 | 宠物/用品发布页面 |
| 商品浏览/搜索 | 筛选列表 + 详情页 |
| 担保交易流程 | 下单 → 支付(模拟) → 确认收货 |
| 管理后台 v1 | 内容审核、用户管理、基础数据统计 |
| 集成测试 + Bug 修复 | 全流程打通 |

### MVP 验收标准

- [x] 用户可注册登录并创建宠物档案
- [x] 用户可发布图文动态并互动
- [x] 用户可发布/浏览/购买商品
- [x] 管理员可审核内容
- [x] 核心流程稳定无阻塞 Bug

---

## 三、Phase 2：社区深化（第11-16周）

**目标**：增强社交属性，提升用户活跃度

### 第11-12周：短视频

| 任务 | 说明 |
|------|------|
| 视频拍摄/上传 | 15-60秒，本地录制或相册选择 |
| 视频转码 + HLS | FFmpeg 后端转码，多码率 |
| 视频播放器 | 自定义播放器（暂停/进度/全屏） |
| 视频互动 | 点赞/评论/分享 |

### 第13-14周：圈子与话题

| 任务 | 说明 |
|------|------|
| 圈子系统 | 创建/加入/退出圈子、圈内动态 |
| 推荐话题 | 热门话题、个性化话题推荐 |
| 话题聚合页 | 话题下的动态瀑布流 |

### 第15-16周：同城社交

| 任务 | 说明 |
|------|------|
| LBS 定位集成 | 高德/百度地图 SDK |
| 同城匹配 | 附近宠物/用户推荐 |
| 约遛/活动 | 发起 → 报名 → 签到 |
| IM 即时通讯 v1 | WebSocket 私信、群聊 |

### Phase 2 里程碑

- 全平台支持短视频内容
- 圈子/话题 DAU 参与率 > 20%
- 同城匹配功能上线

---

## 四、Phase 3：商业闭环（第17-24周）

**目标**：构建完整的商业交易生态

### 第17-19周：完整电商系统

| 任务 | 说明 |
|------|------|
| 真实支付接入 | 支付宝/微信支付 SDK |
| 购物车系统 | 增删改 + 批量结算 |
| 完整订单流 | 待付款/发货/收货/售后退款 |
| 物流查询 | 快递 100 API 对接 |
| 优惠券/促销 | 满减/折扣/新客券 |

### 第20-21周：活体交易合规

| 任务 | 说明 |
|------|------|
| 活体商品审核流程 | 品种/疫苗/健康证明上传 |
| 卖家资质认证 | 身份证 + 繁殖许可证 |
| 担保交易增强 | 资金托管 + 确认放款 |
| 纠纷仲裁系统 | 客服介入 / 证据提交 |

### 第22-24周：服务预约 + 信用评价

| 任务 | 说明 |
|------|------|
| 服务商家入驻 | 资质审核、店铺装修 |
| 服务预约流程 | 选服务 → 选时间 → 支付 → 评价 |
| 评价体系 | 星级 + 图文评价、商家回复 |
| 商家管理端 | 订单管理、服务排期、数据看板 |

### Phase 3 里程碑

- 完整电商闭环（下单/支付/物流/售后）
- 活体交易合规流程上线
- 服务预约 + 评价体系上线

---

## 五、Phase 4：增长与运营（第25-30周）

**目标**：扩大用户规模，提升留存

### 第25-27周：移动端 App

| 任务 | 说明 |
|------|------|
| Flutter 项目搭建 | 跨平台基础框架 |
| 核心功能迁移 | 登录/社区/交易/IM |
| 推送集成 | 极光推送 / 个推 |
| App 性能优化 | 包体积、启动速度、内存 |

### 第28-29周：增长工具

| 任务 | 说明 |
|------|------|
| 邀请裂变 | 邀请码 + 双方奖励 |
| 积分体系 | 签到/任务/消费得积分 |
| Push 运营 | 精准推送、自动化触发 |
| 内容推荐 v2 | 协同过滤算法 |

### 第30周：上线冲刺

| 任务 | 说明 |
|------|------|
| 全链路压测 | QPS 目标、稳定性验证 |
| 安全审计 | 渗透测试、数据加密校验 |
| 应用商店上架 | iOS App Store + 安卓各大市场 |
| 运营后台完善 | 数据看板、活动管理、客服系统 |
| 项目文档归档 | API 文档、部署手册、运维手册 |

### Phase 4 里程碑

- 移动端 App 双平台上线
- 用户增长工具就绪
- 系统稳定运行（99.9% 可用性）

## 七、风险评估

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 活体交易合规问题 | 高 | 高 | 提前咨询法规，设计审核流程，与持牌机构合作 |
| 内容审核压力 | 高 | 中 | 接入第三方审核 API，结合人工抽查 |
| 支付牌照需求 | 中 | 高 | 使用持牌支付机构，不做资金池 |
| 初期冷启动困难 | 高 | 高 | 精准种子用户运营，宠物达人合作 |
| 技术债务积累 | 中 | 中 | 保持 Code Review，定期重构 |
