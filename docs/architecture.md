## 二、技术选型

| 层级 | 技术栈 | 说明 |
|------|--------|------|
| **Web 前端** | Next.js 14 + TypeScript + TailwindCSS + Zustand | SSR + CSR 混合渲染 |
| **移动端** | Flutter 3.x + Dart | 跨平台 iOS/Android |
| **后端语言** | Go (Gin/Kratos) | 高性能，适合微服务 |
| **API 网关** | Kong / Nginx + Lua | 统一入口，限流鉴权 |
| **服务间通信** | gRPC + Protocol Buffers | 高性能内部 RPC |
| **主数据库** | PostgreSQL 16 | 关系型数据（用户/订单等） |
| **内容数据库** | MongoDB 7 | 动态 Feed/评论等非结构化数据 |
| **缓存** | Redis 7 | Session / 热点数据 / 排行榜 |
| **搜索引擎** | Elasticsearch 8 | 宠物/商品/内容全文搜索 |
| **消息队列** | RabbitMQ | 异步任务 / 事件驱动 |
| **对象存储** | 阿里云 OSS / 腾讯 COS | 图片/视频文件存储 |
| **CDN** | 阿里云 CDN / Cloudflare | 静态资源加速 |
| **容器编排** | Docker + Kubernetes | 部署与自动扩缩 |
| **监控** | Prometheus + Grafana + ELK | 链路追踪与日志 |
| **CI/CD** | GitHub Actions + ArgoCD | 自动化构建部署 |

## 三、微服务划分

| 服务 | 职责 | 数据库 |
|------|------|--------|
| **user-service** | 用户注册登录、宠物档案、关注关系 | PostgreSQL |
| **feed-service** | 图文/视频动态发布、信息流聚合 | MongoDB |
| **social-service** | 点赞/评论/转发、互动通知 | MongoDB |
| **community-service** | 圈子/话题、活动组织 | PostgreSQL |
| **marketplace-service** | 商品管理、订单、交易流程 | PostgreSQL |
| **payment-service** | 支付对接、分账、退款 | PostgreSQL |
| **im-service** | 即时通讯、消息推送 | Redis + WebSocket |
| **recommendation-service** | 个性化推荐、搜索 | Elasticsearch |
| **admin-service** | 管理后台、数据统计 | PostgreSQL |

## 四、关键技术决策

### 4.1 微服务 vs 单体
初期如果团队小，建议先用**单体架构**（Go + PostgreSQL），快速迭代验证市场；用户量增长后按服务拆分为微服务。

### 4.2 图片/视频处理
- 图片：上传原图 → OSS → CDN → WebP 自动压缩
- 视频：上传 → FFmpeg 转码 → 多分辨率 HLS 分片 → CDN 分发
- 内容审核：接入阿里云/腾讯云的内容安全 API（黄暴恐识别）

### 4.3 推荐系统 v1 → v2
- v1：基于规则（品种/地区/热度）
- v2：协同过滤 + 向量召回 ( embedding )

### 4.4 安全设计
- 全站 HTTPS，JWT Token 鉴权
- 敏感操作二次验证
- 支付接入合规（持牌支付机构）
- 用户隐私数据加密存储
- 活体交易审核机制 + 信用体系
